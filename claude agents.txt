# RULES

1. Jangan pernah mengirim ulang file penuh.

2. Jangan pernah rewrite seluruh source code.

3. Jangan mengirim file lengkap kecuali saya menulis:

FULL FILE

4. Untuk setiap perubahan gunakan format:

FILE:
[path file]

ACTION:
ADD / MODIFY / REMOVE

SEARCH:
[kode lama]

REPLACE:
[kode baru]

5. Jika perubahan lebih dari satu file:

Tampilkan hanya file yang berubah.

6. Jika perubahan kurang dari 50 baris:

Jangan tampilkan penjelasan panjang.

7. Jangan membuat file baru jika fungsi yang sama sudah ada.

8. Sebelum membuat model, service, API, atau component baru:

Analisa struktur yang ada terlebih dahulu.
